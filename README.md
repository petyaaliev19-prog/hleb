# Хлебушек

`Хлебушек` — веб-сервис предзаказа свежего хлеба и выпечки с самовывозом и оплатой на месте.

## Что уже есть

1. Публичный каталог пекарен для покупателей.
2. Страница отдельной пекарни с ассортиментом, слотами самовывоза и оформлением предзаказа.
3. Оформление предзаказа без онлайн-оплаты.
4. Отдельный кабинет пекарни.
5. Лента заказов в кабинете пекарни.
6. Панель пекарни со сводкой по позициям на день.
7. Смена статусов заказа в панели пекарни.
8. Локальная SQLite база через встроенный `node:sqlite`.

## Текущий стек

1. `Next.js` для приложения.
2. `React` для интерфейса.
3. `node:sqlite` для локального persistence-слоя.
4. `node --test` для юнит-тестов.

## Основные файлы

1. `app/` — страницы и API.
2. `features/` — доменные модули: пекарни, заказы, панель пекарни, кабинет владельца.
3. `components/` — общие UI-компоненты приложения.
4. `server/db/db-repository.js` — работа с SQLite и серверные бизнес-проверки.
5. `lib/` — общие browser/API и форматирующие helpers.
6. `tests/` — юнит-тесты текущей бизнес-логики.
7. `work/hlebushek.db` — локальный файл базы данных.
8. `legacy/standalone-prototype/` — старый автономный прототип, не активное приложение.

## Операционная документация

1. `AGENTS.md` — правила для ИИ-агентов и рабочего процесса.
2. `DOCUMENTATION_RULES.md` — как поддерживать документацию актуальной.
3. `ARCHITECTURE.md` — структура приложения и поток данных.
4. `DATA_MODEL.md` — сущности и бизнес-инварианты.
5. `API_CONTRACTS.md` — активные API-контракты.

## Команды

1. `npm.cmd run dev` — локальный dev-сервер.
2. `npm.cmd test` — запуск тестов.

## Основные маршруты

1. `/` — каталог пекарен для покупателей.
2. `/bakeries/[slug]` — витрина конкретной пекарни.
3. `/owner` — кабинет пекарни.
4. `/owner/bakeries/[slug]` — управление карточкой и операционной сводкой пекарни.
5. `/owner/orders` — лента заказов для пекарни.
